# 首页内容区视觉优化

这条用于测试首页最新 Todo 展示。Todo 在第一版里是个人路线图，不是网页端任务管理器。

## 目标

- 让首页内容区能承接首屏，而不是像临时拼接的链接区。
- Blog、Wiki、Todo、Reflection 都有明确入口和信息价值。
- 移动端保持单列可读，不制造横向滚动。

## 验收口径

| 检查项 | 目标 |
| --- | --- |
| 状态 | 使用只读标签展示 |
| 优先级 | 明确但不过度强调 |
| 内容来源 | 只来自 `content/todo/` |

## 备注

后续如果要调整状态，需要修改 Markdown 文件并重新构建；网页端不会持久化任何任务状态。
